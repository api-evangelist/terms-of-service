Yesterday on twitter I made a comment criticizing the practice of putting an API key in a query string parameter.  I was surprised by the amount of attention it got and there were a number of responses questioning the significance of my objection.  Rather than try and reply in 140 character chunks, I decided a blog post was in order.Most of the comments were security related, — Pascal VanVlaenderen (@VanVlaenderenP) December 17, 2014 It is true that whether the API key is put in the URL or in the Authorization header.They are both going to be sent over the wire in clear text.
